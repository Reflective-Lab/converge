# v3.8 Foundation

v3.8 is a foundation release. Prefer fewer, stronger concepts over wider
surface area. The release should be boring to depend on for a long time.

## Core Ideas

1. **Contracts get the real names.**
   The semantic contract owns the clean domain name. Implementation crates add
   adapter words such as `http`, `openai`, `brave`, `surreal`, or `runtime`.
   `converge-provider` is the provider contract; generic provider/tool
   implementations live in Manifold.

2. **Suggestors only propose.**
   `AgentEffect` is a finalized proposal-only output. Incremental construction
   belongs to `AgentEffectBuilder`, not mutation on the finished effect.

3. **Promotion authority is not a Cargo feature.**
   A publishable feature is observable by downstream crates through feature
   unification. Authority construction must move behind a real crate boundary,
   not a public feature flag.

4. **External I/O lives outside the kernel.**
   Provider adapters, tool integrations, HTTP clients, API keys, network
   retries, service SDKs, and feed/search/fetch implementations belong outside
   the Converge kernel repository or in clearly adapter-qualified crates.

5. **Implementation-heavy capabilities are extensions.**
   Providers, analytics, policy engines, storage adapters, and knowledge stores
   implement Converge contracts. Their heavy dependencies and independent
   release cycles should not define the foundation release cadence.

6. **Semantic values carry their meaning.**
   Confidence, score, ratio, limit, timeout, status, hash, identifier, and URL
   values use typed domains. Property tests are not a substitute for impossible
   states.

7. **Provider and tool contracts are ports.**
   The stable contract crate owns traits, request/response DTOs, and capability
   vocabulary. Adapter crates implement those ports. The contract must not
   depend on `converge-pack` to borrow unrelated value types.

8. **Security, coverage, and performance are release artifacts.**
   v3.8 ships with reproducible commands and stored baselines for dependency
   audit, security regression checks, test coverage, Criterion performance, and
   soak behavior.

9. **The KB is part of the product.**
   Stale crate names, moved responsibilities, old release plans, and weak facts
   are removed or demoted. The KB should make the core concepts above
   hard to miss.

## 3.8 Workstreams

- Seal public construction boundaries: `AgentEffect`, `Fact`, promotion
  metadata, and compile-fail misuse proofs.
- Harden external fetch/feed/search against SSRF, oversized responses, unsafe
  redirects, and unbounded limits.
- Decide and document the provider/tool extraction path.
- Classify provider, analytics, policy, knowledge, connector, storage, and
  solver implementation crates as foundation, extension, or extraction
  candidates.
- Establish naming rules for contract crates and implementation crates.
- Add `security-audit`, coverage, performance-profile, and soak recipes.
- Run KB sanity checks against code and remove stale `provider-api`,
  `ortools-sys`, and moved-provider assumptions where they are no longer true.

## Extraction Progress

- [x] **mnemos** — knowledge extracted from `crates/knowledge` to
  `~/dev/extensions/mnemos` on 2026-05-05. Foundation no longer ships vector
  storage, agentic memory, or learning implementations.
- [x] **prism** — analytics extracted from `crates/analytics` to
  `~/dev/extensions/prism` on 2026-05-05. Foundation no longer ships
  Polars/Burn ML pipelines or analytics packs.
- [x] **manifold provider/tool adapters** — Converge-side dependency
  break is in place: `ChatBackendSelectionConfig` lives in
  `converge-provider`, `ProviderSelectionSuggestor` lives in
  `converge-kernel`, and neither `converge-runtime` nor `converge-kernel`
  depends on an adapter crate. The host-supplied `ChatBackendRegistry`
  boundary also lives in `converge-provider`. Generic LLM, search/fetch/feed,
  embedding/reranker/vector, and OpenAPI/GraphQL tool adapters have moved to
  Manifold; `converge-provider-adapters` has been removed. Implementation plan:
  [[Planning/Manifold Provider Tool Migration]].
- [x] **manifold storage adapters** — concrete local/S3/GCS object-store
  builders, SurrealDB/LanceDB experience-store adapters, and the generic
  LanceDB vector adapter live in `~/dev/extensions/manifold-adapters` on
  2026-05-05.
  Foundation `converge-storage` and `converge-experience` keep contracts and
  in-memory support only.
- [x] **arbiter** — Cedar policy engine extracted from `crates/policy` to
  `~/dev/extensions/arbiter` on 2026-05-05. Foundation `converge-pack` keeps
  the gate trait and authorization vocabulary; Cedar wiring, policy
  suggestors, and ed25519 delegation tokens live in arbiter.
- [x] **atelier** — `crates/domain` and `examples/*` extracted to
  `~/dev/atelier` on 2026-05-05 as a single workspace of worked exemplars
  (showcase, not extension). Will gather demos from organism and axiom over
  time. The `analytics-packs` example was deferred pending prism rewiring.
- [x] **ferrox** — relocated from `~/dev/work/ferrox` to
  `~/dev/extensions/ferrox` on 2026-05-05 to match the extension naming and
  one-way dependency rule.
- [x] **embassy** — connector ports live in `~/dev/extensions/embassy`.
  Embassy is for source-specific business integrations where the external
  party is part of the semantic contract, such as LinkedIn. Generic
  capability adapters still belong in manifold.
- [x] **storage boundary** — storage is a port, not a new plugin category.
  Runtime persistence is host-wired; capability storage/search is routed
  through backend requirements. Reusable database adapters belong in
  extensions, while Runway owns operational assembly. The reusable adapters
  are extracted to Manifold. See [[Architecture/Storage Boundary]].

## Downstream Sequencing

ADR-006 must land before Organism builds new Fact-adjacent bridge work.
Organism currently has no `kernel-authority` imports, but the planned Truth
Document -> IntentPacket bridge and `Runtime.resolve_and_admit_truth()` are
authority-adjacent. Building them against the current feature-gated authority
path would make the bridge born-deprecated.

Known downstream priority:

- **Block first:** land the real authority crate boundary before Organism's
  Truth bridge work.
- **Migrate production:** Helms `application-storage` has the load-bearing
  production Fact construction exposure.
- **Defer research:** newspaper engagement call sites can wait for the new
  boundary unless they become production dependencies.
- **Bidirectional experience:** Organism issue #12 needs user-side experience
  events to become planning priors. Converge now exposes
  `UserApprovalRejected`, `UserCorrection`, and `UserBoundaryAdjusted` beside
  the existing approval/override variants, and `recall_from_store` consumes
  all five user variants into success, anti-pattern, or runbook candidates.
