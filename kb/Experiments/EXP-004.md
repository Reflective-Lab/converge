# EXP-004: Memory Stability Under Sustained Load

**Hypothesis:** The convergence engine exhibits stable memory usage under sustained load: running 1000+ sequential engine cycles should not cause unbounded heap growth. RSS (resident set size) growth should remain < 50% per 100 iterations.

**Rationale:** In production, agents may run convergence cycles thousands of times. Uncontrolled memory leaks could cause:
- System memory exhaustion and OOM kills
- Increasing latency as GC pressure mounts
- Instability in long-running services

Memory stability is essential for production reliability.

## Test Approach

**File:** `crates/core/tests/soak.rs:soak_memory_stability`

Measures RSS (resident set size) before and after repeated engine cycles:

```rust
#[tokio::test]
#[ignore = "soak"]
async fn soak_memory_stability() {
    let iterations = env::var("SOAK_ITERATIONS").unwrap_or("100".to_string()).parse().unwrap_or(100);
    
    let rss_before = read_rss_kb().unwrap_or(0);
    
    for _ in 0..iterations {
        let mut engine = Engine::new();
        engine.register_suggestor(SeedSuggestor::new("seed", "test data"));
        let _ = engine.run(Context::new()).await;
    }
    
    let rss_after = read_rss_kb().unwrap_or(0);
    let growth_mb = (rss_after - rss_before) / 1024;
    
    // Threshold: growth < 50% per 100 iterations
    let threshold = (iterations as f64 / 100.0) * 50.0;
    assert!(growth_mb < threshold);
}
```

## Metrics

| Metric | Target | Rationale |
|--------|--------|-----------|
| RSS growth per 100 iterations | < 50 MB | Allows for buffers, caches; flags unbounded growth |
| Stability over 1000 cycles | Linear or sublinear growth | No exponential blow-up |
| GC pressure | Stable | No increasing pause times |

## Execution

**Local (detailed profiling):**
```bash
SOAK_ITERATIONS=1000 cargo test --test soak soak_memory_stability -- --include-ignored --nocapture
```

**CI (quick check):**
```bash
SOAK_ITERATIONS=100 cargo test --test soak soak_memory_stability -- --include-ignored
```

## Memory Tracking Method

Uses `/proc/self/status` (Linux only) to read VmRSS:
- **Why RSS**: Includes page cache and shared memory, reflects actual system impact
- **Fallback**: Returns None on macOS/non-Linux; test skips threshold check but still runs
- **Resolution**: 4KB page granularity (typical), ±5KB noise acceptable

## Regression Thresholds

- **Alert:** RSS growth > (iterations / 100 * 50) MB
- **Critical:** RSS growth > (iterations / 100 * 100) MB (100% of threshold)
- **Acceptable variance:** ±10 MB (measurement noise)
