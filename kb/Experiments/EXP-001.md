# EXP-001: Engine Single-Cycle Latency Target

**Hypothesis:** The convergence engine can complete a single cycle with one seed suggestor in under 1 millisecond (p99 latency).

**Rationale:** Single-cycle latency is the floor for all multi-cycle convergence. This establishes the baseline performance ceiling for the Engine. Any degradation here cascades to all downstream convergence paths.

## Acceptance Criteria

- **p99 latency < 1000 µs** (1.0 ms) on a clean, unloaded system
- **Consistent across runs** — same hardware produces ±5% variance in p99
- **No regression** — p99 latency must not increase > 10% from previous baseline
- **Deterministic output** — all runs produce identical fact counts

## Test Case

**File:** `crates/core/benches/engine_bench.rs:bench_engine_single_cycle`

```rust
fn bench_engine_single_cycle(c: &mut Criterion) {
    c.bench_function("engine_single_cycle", |b| {
        b.iter(|| {
            let rt = tokio::runtime::Runtime::new().expect("create runtime");
            rt.block_on(async {
                let mut engine = Engine::new();
                engine.register_suggestor(SeedSuggestor::new("seed-1", "test data"));
                engine.run(Context::new()).await.expect("should converge")
            })
        });
    });
}
```

**Execution:** `cargo bench -p converge-core -- engine_single_cycle`
