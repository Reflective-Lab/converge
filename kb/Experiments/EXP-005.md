# EXP-005: Engine Resilience Under Chaos

**Hypothesis:** The convergence engine gracefully handles adversarial conditions including panic injection, malformed proposals, and latency variance without losing data or producing corrupted output.

**Rationale:** Production systems encounter failures, timing issues, and malformed input. The engine must be resilient to these conditions to maintain system stability and data integrity.

## Acceptance Criteria

- **Panic containment** — Panic in one suggestor does not crash the entire engine
- **Malformed rejection** — Proposals with null bytes, empty IDs, or oversized content are rejected, not crash
- **Latency independence** — Output is deterministic regardless of suggestor latency
- **Recovery** — Engine can complete a convergence cycle even when some suggestors fail
- **No data corruption** — Facts added before a failure remain valid and consistent

## Test Cases

**File:** `crates/core/tests/chaos.rs`

### Chaos Vectors

| Test | Purpose | Condition |
|------|---------|-----------|
| `chaos_panic_on_cycle_2_others_complete` | Panic isolation | Panic on cycle 2; 4 safe seeds should complete |
| `chaos_malformed_overconfident_proposal` | Confidence validation | Reject proposal with confidence > 1.0 |
| `chaos_malformed_null_byte_id` | ID validation | Reject ID with embedded null byte |
| `chaos_malformed_null_byte_content` | Content validation | Reject content with embedded null byte |
| `chaos_malformed_empty_id` | ID validation | Reject empty ID string |
| `chaos_malformed_whitespace_id` | ID validation | Reject whitespace-only ID |
| `chaos_malformed_giant_content` | Size validation | Reject content > 1MB |
| `chaos_latency_variance_preserves_determinism` | Timing independence | Same output with 0ms and 10ms injected latency |
| `chaos_mixed_panic_and_malformed` | Combined chaos | Handle panic + malformed proposals together |
