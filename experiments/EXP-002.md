# EXP-002: Rust optimization self-sufficiency vs OR-tools FFI
**Date:** 2026-04-22
**Status:** confirmed
**Hypothesis:** Converge already has the mathematical tools it needs in pure Rust for its current 21 packs, and OR-tools FFI is only needed for LP/MIP and advanced CP-SAT — gaps that can be closed incrementally with HiGHS bindings rather than full OR-tools dependency.
**Falsification criterion:** If any of the 21 current packs require OR-tools to produce correct/optimal results, or if HiGHS cannot cover the LP/MIP gap, the hypothesis fails.
**Method:** Audit current algorithm coverage, identify gaps, compare with OR-tools capabilities, assess Rust ecosystem alternatives.

## Audit: What Converge Already Has (Pure Rust)

| Domain | Algorithm | Quality |
|--------|-----------|---------|
| Assignment | Hungarian (O(n³)) | Exact, optimal |
| Assignment | Auction algorithm | Exact, fast for sparse |
| Graph | Dijkstra shortest path | Exact |
| Graph | Push-Relabel max flow | Exact, O(V²E) |
| Graph | Min-cost flow | Exact |
| Graph | Hopcroft-Karp bipartite matching | Exact, O(E√V) |
| Knapsack | Dynamic programming (0-1) | Exact |
| Set cover | Greedy + local search | ln(n) approximation |
| Scheduling | List scheduling, interval scheduling | Heuristic |
| CP | SAT-based via Varisat (optional) | Functional, limited |

| Bin packing | First-Fit Decreasing | Heuristic |
| Graph partition | Greedy Kernighan-Lin | Heuristic |

**Verdict:** All 21 packs run without OR-tools (`default` features). The FFI is optional and unused in production.

## What OR-Tools Provides That's Hard to Replicate

### 1. CP-SAT solver — THE crown jewel
- Hybrid: SAT + LP relaxation + Large Neighborhood Search + automatic parallelism
- Dominates MiniZinc Challenge benchmarks
- Nothing comparable exists in Rust. Varisat is a CDCL SAT solver — no hybrid CP capabilities.
- **Gap: massive.** Building an equivalent is a multi-year effort.

### 2. LP/MIP solvers (GLOP, SCIP, CBC wrappers)
- Value is solver-switching behind uniform API
- **Gap: closeable.** HiGHS (open-source, top-tier MIP) has Rust bindings. `good_lp` crate abstracts over multiple backends.

### 3. VRP library
- Production-grade: time windows, capacity, pickup/delivery, breaks, multi-depot
- Thousands of engineer-hours of domain heuristics
- **Gap: large but not blocking.** Converge's VRP pack uses nearest-neighbor heuristic — adequate for current use cases.

### 4. Network flows
- Well-optimized but **already covered** by Converge's native implementations.

## Rust Ecosystem: The Incremental Path

| Need | Crate | Build complexity | Quality |
|------|-------|------------------|---------|
| LP/MIP | `good_lp` + `highs` | cmake (manageable) | Top-tier open-source MIP |
| LP/MIP | `coin_cbc` | system lib | Mature, slower than HiGHS |
| Conic/QP | `clarabel` | **Pure Rust** | High quality, no integer support |
| MIP | `russcip` | C build | SCIP is powerful, build is friction |
| SAT | `varisat`, `splr` | Pure Rust | Decent CDCL, no CP-SAT hybrid |
| CP | — | — | **Nothing production-grade in Rust** |

### Recommended incremental strategy

**Phase 1 — Now (no new deps):** Ship what exists. All 21 packs work with pure Rust heuristics. EXP-001 confirmed correctness.

**Phase 2 — When LP/MIP needed:** Add `good_lp` + `highs` as optional feature. Single cmake dependency, builds on all platforms. Covers budget allocation, capacity planning, facility location with exact solutions instead of greedy heuristics.

**Phase 3 — When CP needed:** Two options:
- (a) OR-tools FFI (already scaffolded in `ortools-sys`) — heavy build, but gets CP-SAT
- (b) Build a mini CP solver on top of Varisat with LNS — lighter, fits Converge's pack model, won't match CP-SAT quality but may suffice for scheduling/rostering packs

**Phase 4 — If VRP gets serious:** Either enhance native heuristics (add 2-opt, or-opt, relocate moves) or pull in OR-tools VRP. Cross that bridge when needed.

## Evidence

- `cargo build` succeeds with default features (no FFI, no SAT) — all 21 packs compile
- EXP-001 confirmed 37/37 reference tests pass with native algorithms
- `ortools-sys` exists but is gated behind `ffi` feature — zero impact on default builds
- HiGHS is the COIN-OR successor, actively developed, Apache-2.0 licensed, top MiniZinc LP/MIP results

## Outcome
**Confirmed.** Converge is self-sufficient in pure Rust for its current scope. The gaps are:

1. **LP/MIP (closeable):** `good_lp` + HiGHS — add when a pack needs exact LP/MIP instead of greedy approximation
2. **CP-SAT (massive gap):** No Rust equivalent. Use OR-tools FFI or build a limited hybrid on Varisat
3. **VRP (not blocking):** Current heuristic is adequate; enhance incrementally

The elephant doesn't need eating. The 80% is already done in Rust. The remaining 20% can be added as optional features, one piece at a time.
